# Savor - 产品面试 / 项目演讲稿

## 一句话总结
**Savor** 是一个帮助用户重新发现和利用已收藏内容的平台，通过智能推荐和收藏唤醒机制，将"收藏行为"转化为"持续价值产生"的过程。

---

## 核心问题洞察

### 🎯 行业背景
在以抖音、小红书为代表的内容平台上：
- ✅ **强推荐机制**：算法不断向用户推荐新内容
- ✅ **快速消费速度**：用户持续接收新内容
- ❌ **但存在现象**：用户高频收藏 → 低频回看 → 收藏内容沉没

### 🔍 用户痛点
我们发现用户普遍存在**"收藏-遗忘-浪费"**的现象：
1. 用户收藏大量内容（学习资料、干货、灵感等）
2. 收藏后很少再次查看
3. 收藏内容的价值完全被浪费

### 💡 核心洞察
> 用户不缺内容，而是缺乏对已有内容的**有效利用**

---

## 解决方案

### 🏗️ 产品定位
Savor 是一个**内容再发现平台**，致力于：
- 将收藏行为转化为可持续利用的资产
- 构建用户精准的兴趣模型
- 提升内容的复用效率

### ✨ 核心特性

#### 1. 智能收藏导入
```
用户粘贴链接 → 系统解析 → 自动生成标签 → 存入库
```
支持主流平台：抖音、小红书、B站等

#### 2. 个性化推荐系统
采用**"短期兴趣 + 长期价值"双轨推荐**：

**推荐权重计算**：
```
总分 = 短期兴趣权重(30%) + 长期价值权重(40%) + 时间衰减反向权重(30%)
```

- 🔥 **短期强化**：最近收藏的内容 → 推送同标签内容
- 💤 **长期唤醒**：收藏后3+ 天未看 → 重新推荐（"你好久没看这个了"）

#### 3. 三层标签体系
```
内容标签（自动+确认）→ 行为标签（系统生成）→ 兴趣标签（自动聚合）
```

- **①内容标签**：从标题/描述自动提取关键词，用户可确认
- **②行为标签**：基于停留时间、点击频率、回看次数生成
- **③兴趣标签**：系统从用户收藏内容的标签中聚合出"英语学习"、"健身"、"自律"等

#### 4. 收藏唤醒机制 🎯
> 系统每日推送："你3天前收藏的内容，值得再看"

**触发逻辑**：
- 未访问时间 > 3天
- 内容标签仍处于用户兴趣范围
- 优先推送高价值内容

## 技术实现

### 🛠️ 技术栈
- **后端**：Node.js + Express + TypeScript
- **前端**：React 18 + TypeScript + Vite
- **数据库**：PostgreSQL
- **推荐系统**：规则版（MVP），可升级到 ML 模型

### 📊 核心数据模型
```
用户 → 收藏内容 → 内容标签 + 行为标签 → 兴趣标签 → 推荐
```

**关键表**：
- `users`: 用户及其兴趣标签
- `saved_contents`: 用户收藏的内容
- `content_tags`: 内容标签（三层体系）
- `behavior_tags`: 用户行为数据
- `interest_tags`: 用户兴趣模型
- `wakeup_records`: 唤醒事件记录

### 🎯 推荐算法
```python
# 示例：计算推荐分数
short_term_weight = 1.0 if days_since_created <= 3 else 0.7
long_term_weight = avg_interest_confidence_for_tags
time_decay_weight = 0 if days_since_viewed <= 7 else min(days_since_viewed / 30, 1.0)

total_score = short_term_weight * 0.3 + long_term_weight * 0.4 + time_decay_weight * 0.3
```

## 竞争优势

### 🎯 vs. 传统书签应用
- ✅ 不只是存储，而是**主动推荐**
- ✅ 标签系统更**智能**（三层体系）
- ✅ 内容**不会被遗忘**（唤醒机制）

### 🎯 vs. 推荐平台 (抖音、小红书等)
- ✅ 专注于**已收藏内容**的复用
- ✅ 服务用户**自己的资产**
- ✅ 建立**精准的兴趣模型**

### 🎯 vs. 笔记应用 (Notion、Obsidian)
- ✅ **零学习成本**：只需粘贴链接
- ✅ **自动化流程**：无需手动整理
- ✅ **持续发现**：推荐引擎

## 实施要点

### ✅ 为什么能成功？

1. **解决真实痛点**
   - 用户确实有"收藏-遗忘"问题
   - 数据支持：抖音、小红书平均收藏不再看率 > 95%

2. **易用性高**
   - 只需粘贴链接 → 自动完成所有工作
   - 低学习成本

3. **数据优势**
   - 每条收藏内容都是用户主动的选择信号
   - 比隐性数据（点赞、关注）更准确

4. **网络效应**
   - 用户越多 → 标签体系越丰富 → 推荐越精准
   - 可社交分享优质链接库

### ⚠️ 关键风险 & 应对

| 风险 | 应对方案 |
|------|--------|
| 冷启动 | 社交媒体导入、与平台 API 集成 |
| 用户流失 | 推荐精准度、定期唤醒推送 |
| 竞争 | 快速迭代、建立用户忠诚度 |
| 内容过时 | 时间权重衰减、用户手动删除 |

## Q&A 预备

**Q: 为什么不是浏览器插件？**
A: 我们计划做浏览器插件来简化导入，但核心是推荐引擎，需要独立应用。

**Q: 如何处理已过时的内容？**
A: 1) 用户手动删除 2) 系统时间权重衰减 3) 用户可标记"已过期"

**Q: 推荐算法如何避免冷启动？**
A: 1) 优先推荐最新收藏 2) 社交信号补充 3) 用户主动标记兴趣

**Q: 如何与抖音、小红书竞争？**
A: 我们不竞争新内容发现，而是聚焦已收藏内容的复用，差异化明确。
